## Result (bottom line)

**NEGATIVE.** The register's own alphabetical "mit Register" baptism index —
read leaf-by-leaf, every surname, A→Z — contains **no Konze-pool surname of any
spelling anywhere in 1830–1867**, therefore none in the 1833–1841 window:

- **K section** (Bild **1062**R + **1063**) — no **Konze / Kontze / Konsen / Konzen**.
  K surnames present: Köster/Koester, Kurth, King, Köhler, Kaufmann, Kirschknecht,
  Kämpfe, Kümmel, Kümpel, König, Kunst, Kraft, Kluthe, Kindbe, Bury.
- **G section** (Bild **1059** + **1060**L; duplicate G leaf at **1067**) — no
  **Gansen / Gonsen / Gonson / Gannsen / Gans / Ganß**. G surnames present:
  Gerland (the dominant native clan), Götze, Grabe, Gante, Gödchen, Gemmern,
  Garte, Grek/Greb, Gödecke.
- **I / J section (Jungkonze / Jungkontze):** **no I/J leaf exists in the index
  at all** — the alphabet runs …H (1060–61) → [shuffled D] → K (1062–63) → M
  (1064) with no I/J page, and no "Jung-" form appears on any leaf ⇒ no Jungkonze.
- **C (Conson / Conze):** the alphabet jumps **B → D** (no separate C page), so
  any C-variant would file under K — already cleared.

Only one regex "hit" in the whole sweep, a **false positive**: *"Blankenberg,
Kons. Wilh. August"* (Bild 1055, B section) — the surname is **Blankenberg**;
"Kons." is an abbreviation in the given-name field, not the surname Konze.

This **independently confirms and sharpens** the 26-Jun baptism-window sweep
(which had logged the index conclusion but no image numbers or permalink) and
**corrects the old memory note** that placed "K = Bild 1062R": the index is in
fact bound **out of order with duplicate leaves**, so 1062 is a **D | K** spread,
not a clean alphabetical opening.
