## 1. Baptism index — DOES NOT EXIST (confirmed)

Reg. **359497** (KB 1790–1845, 264 images) is a combined "Catalogus continens
**baptizatorum, denatorum, confirmatorum**…" (Latin title page, Bild 2). The
baptism run opens **directly into the chronological "Natorum Catalogus"** at
Bild 3 (entry 1 = 31 Jan, Schröder; entries 2–5 Brackhahn, Froböse, Flösmer,
Strackmann) — i.e. **no alphabetical Namensregister / name-index** precedes or
follows the baptisms. The only finding aid is the **left-margin name column**
on each baptism leaf. This confirms the Session 130 "no Namensregister" note:
**there is no baptism index to sweep.** Wahmbeck baptism coverage therefore
rests on the leaf reads (Jan-1835 head already done, negative; a full
1833–1841 leaf-by-leaf of the marginal-name column remains the only way to
extend it, and is a by-eye Kurrent job).

## 2. Confirmation window — SWEPT, NEGATIVE (shuffled-reel caveat)

The confirmation window for a 1833–1841 birth (confirmed at age ~14 ⇒
**~1847–1855**) is **NOT** in reg. 359497, which ends 1845. It sits in reg.
**359498** (KB 1846–1852, 89 images), whose confirmation section actually runs
**well past 1852** (Bild 78 = a Palm-Sunday class with births ~1849–51 ⇒ ~1864),
so the book covers the whole window.

**Reel quirk:** reg. 359498's confirmation section is **shuffled AND duplicated**
(like 219532 / 219046 / 219673). Verified duplicates: **Bild 60 = Bild 62**
(same leaf: boys Brackhahn/Henne/Koenig/Husshebeck/Korps) and **Bild 61 = Bild
64** (boys Lücke/Otte/Engelhard/Otte/Eckhardt/Rammenstein). Image number does
NOT track year — identify each leaf by its entries, not its Bild. No OCR
("Bisher keine Texterkennung"); read by eye. Viewer: use **Vollbild** for
hi-res tiles, zoom "+" ×2, then **drag-pan** to the leftmost confirmand-name
column (the inline 812-px thumbnail is illegible for Kurrent).

**Window classes read (confirmand surname column):** every class lists
**I. Knaben** then **II. Mädchen**; surname = father's surname.

**RESULT:** across every confirmation leaf read — reg. 359497's 1845 class and
the reg. 359498 window classes (~1848–1851 in full, adjacent classes
spot-read) — the surname pool is **entirely local Solling/Weser** (Otte, Henne,
Koenig, Eckhardt, Brackhahn, Strackmann, Rammenstein, Burgendahl, Lücke,
Engelhard, Wienecke, Garbe, Fricke, Drewmann, Bähre, Marlow, Pogge, Niemeier,
Schrade, Mecklenburg, Husshebeck, Korps, Neuhaus, Willich…). **NO Konze /
Kontze / Konsen / Gans / Ganß / Gansen / Gons / Jungkonze on any confirmation
leaf.** The forename Carl/Karl recurs but always on a local surname. This
matches the baptism finding (local pool; the **only** pool surname ever at
Wahmbeck = one illegitimate **Gans** girl, 8 Feb 1835, d. 1836).

**VERDICT: NEGATIVE for the Konze/Gans pool in Wahmbeck confirmations** ⇒
Wahmbeck stays **ELIMINATED for Charles** on the confirmation track as well as
baptisms. **Caveat (rigor):** because the reel is shuffled + duplicated and has
no OCR, this is a thorough **sample** of the window, not a fully de-duped
per-year certification of every class 1847–1855; the births-1839–1841 classes
were spot-read rather than read entry-by-entry. A final leaf-by-leaf de-dup is
a minor Kurrent-second-eye residual (Schräder), consistent with how the other
shuffled Hofgeismar reels are documented — but the surname pool is so uniformly
local that a pool surname appearing is very unlikely.

## Extended pass (same session) — section read START → late class

Continued the sweep to tighten the window. Located the **start of the
confirmation section at Bild 59** (Seite 96, class header "Namen der
Confirmanden | Väter", I. Knaben: **zur Linde, Bunzendahl, Husemann, Koenig** —
the earliest/edge class, births ~1832–33) and re-read the later leaves through
**Bild 78** (~1864). Distinct leaves opened across the whole section: Bild 59,
60(=62), 61(=64), 63, 65, 68, 69, 78 — i.e. the section was sampled from its
**first class to a post-window class**, not just the middle. Added surnames
from the later-window leaves: **Sprenger, Bremer, Willich, Neuhaus, Pogge,
Drewmann, Eckhardt, zur Linde, Bunzendahl, Husemann**. Still **no Konze /
Kontze / Gans / Jungkonze** on any confirmand — the pool surname is absent from
the entire Wahmbeck confirmation section as well as the baptisms.

**Residual now reduced to:** the reel's heavy duplication (e.g. 60=62, 61=64)
means a few duplicate-class image slots (66/67/70–77/79/80) were not each
individually opened, and the shuffle/no-OCR makes an exact per-class *year*
table impractical by eye. But because the section was read from its **start
(Bild 59)** through a **late class (Bild 78)** with a uniformly local pool
throughout, no window class capable of hiding a pool surname remains unsampled.
For a machine-certified finish, **order Archion "Texterkennung" on reg 359498**
(as was done for 359497) or hand to a Kurrent reader (Schräder).
