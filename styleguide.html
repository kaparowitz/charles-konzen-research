Style Guide

Single source of truth for color, type, components, and status semantics · CSS tokens in assets/theme.css

The gold accent (--brand) is reserved for site-wide brand and primary CTA only. It used to do double duty as "open status" and "Charles in America accent" — both now use distinct colors so brand stays singular.

Five section colors at roughly 50% saturation / 45% lightness — coordinated rather than independently chosen. Each section's deep-dive pages, hero accent, and cross-link tiles use its own color.

8 · Hover behavior (canonical)

Single transition rule applies to every clickable surface site-wide. Defined in assets/theme.css:

--hover-lift:    translateY(-1px);
--hover-shadow:  0 2px 10px rgba(0,0,0,0.10);
--transition:    all 0.15s ease;

9 · How to use these tokens

Every shared CSS file pulls from assets/theme.css via var(--token-name). To re-theme the entire site, edit one file. To add a new section accent, add a new --section-NAME pair to the root and reference it in the new section's tile rules.

  • Don't hard-code hex colors in new code — always go through the variables.
  • Don't introduce new accent colors without adding them to assets/theme.css.
  • For status overrides on tiles: use the --status-* variables, never the brand or section variables.
